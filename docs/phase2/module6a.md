## 第1课：AI会议纪要 —— 从录音到行动清单

**时长**: 约90分钟 | **类型**: 实操为主

### 核心概念

会议纪要的本质不是"记录谁说了什么"，而是**将讨论转化为行动**。一场好会议结束后，每个参会者都应该清楚：**我接下来要做什么？截止日期是什么？**

AI会议纪要工具的黄金工作流：

```
录音 → 语音转文字 → AI提炼要点 → 结构化纪要 → 生成待办 → 群发确认
```

### 2026年主流AI会议工具对比

| 工具 | 转写准确率 | 亮点功能 | 免费额度 | 适用场景 |
|------|----------|---------|---------|---------|
| **飞书妙记** | 96%+ | 自动分章节、关键词高亮、发言人分离、一键生成待办 | 飞书用户免费 | 飞书生态用户首选 |
| **通义听悟** | 95%+ | 实时翻译、章节速览、PPT导出、多语言支持 | 每日5小时免费 | 通用场景、跨平台 |
| **腾讯会议AI** | 94%+ | 实时字幕、智能摘要、待办自动提取 | 专业版含AI | 腾讯会议用户 |
| **讯飞听见** | 97%+ | 方言识别、行业术语库、多语种转写 | 每日1小时免费 | 正式会议、发布会 |
| **Otter.ai** | 93%+（英文） | 实时协作笔记、Slack集成 | 每月300分钟 | 英文会议、国际团队 |

### 动手实操：从一段录音到一份完美纪要

#### Step 1：录音与转写（15分钟）

**方法A：飞书妙记**（推荐）
1. 打开飞书 → 左侧导航栏点击"妙记"
2. 点击"上传音视频" → 选择你的会议录音文件
3. 等待AI自动转写（通常5-10分钟）
4. 转写完成后，AI会自动分章节、识别发言人

**方法B：通义听悟**
1. 打开 [通义听悟](https://tingwu.aliyun.com/)
2. 上传音频文件或粘贴链接
3. 选择语言（中文/英文/中英混合）
4. 等待转写完成

**方法C：腾讯会议AI纪要**
1. 在腾讯会议中开启"云录制"和"AI纪要"
2. 会议结束后，在"历史会议"中找到AI生成的摘要

#### Step 2：AI提炼要点与结构化（20分钟）

将转写文本导出，用AI工具做深度提炼：

```text
你是一位专业的会议纪要撰写专家。以下是一场[会议主题]的完整转写文本。

请帮我把这份会议记录转化为一份高质量的会议纪要：

[粘贴转写后的全文]

请按以下结构输出：

**一、会议基本信息**
- 时间/地点/参会人/主持人/记录人

**二、核心议题与讨论要点**（按议题分组）
- 议题1：[议题名称]
  - 讨论要点（3-5条，每条一句话）
  - 关键分歧（如有）
  - 最终结论

**三、决策事项**（编号列出所有在会上达成共识的决策）

**四、行动待办清单**（用表格呈现）

| 序号 | 待办事项 | 负责人 | 截止日期 | 优先级 |
|------|---------|--------|---------|--------|
| 1 | | | | |

**五、遗留问题**（本次会议未解决、需要后续跟进的问题）

**六、下次会议预告**
- 建议时间
- 建议议题

要求：
1. 语言简洁专业，能用短句不用长句
2. 避免记录"谁说了什么"，转为"讨论了什么、决定了什么"
3. 待办事项必须具体可执行，不能模糊（如不能说"跟进XX"，要说"在XX平台完成XX操作并反馈结果"）
4. 标注信息不明确的地方为"[待确认]"
```

#### Step 3：AI行动项追踪（15分钟）

会议纪要有了，但真正拉开差距的是"跟进"：

```text
基于以上会议纪要，请帮我生成一份"行动项追踪表"：

格式要求：
1. 用表格形式，方便复制到飞书多维表格/Excel
2. 包含以下列：序号、待办事项、负责人、截止日期、状态（未开始/进行中/已完成/已延期）、预计耗时、风险标签
3. 对每个待办事项做风险评估：如果是"高依赖"（需要等别人先完成）或"高不确定性"的任务，标注出来
4. 按优先级排序：P0（本周必须完成）→ P1（本月完成）→ P2（有时间就做）
5. 生成一个"建议的跟进节奏"：什么时间点检查哪些任务的进度
```

#### Step 4：会议纪要分发与确认（10分钟）

```text
请基于以上会议纪要，帮我写一段发送给参会人员的消息/邮件：

内容要求：
1. 一句话总结会议核心成果
2. 列出你需要对方确认/补充的部分
3. 附上待办清单中与对方相关的部分（高亮）
4. 语气：协作型、鼓励型，不是命令型

同时生成3个版本：
- 版本A：群聊消息版（适合飞书/企业微信群发，简短）
- 版本B：邮件版（适合正式群发邮件，有标题有正文）
- 版本C：1对1私聊版（适合单独发给某位负责人，更亲切）
```

### 避坑指南

| ❌ 常见错误 | ✅ 正确做法 |
|-----------|-----------|
| 照搬AI转写，不审校 | AI转写会有错别字和术语错误，必须人工过一遍 |
| 纪要太长，没人看 | 控制在1-2页，重点突出"决策+待办" |
| 纪要只记录"说了什么" | 要记录"决定了什么"和"谁接下来做什么" |
| 不标注信息来源 | 如果某条信息不确定，标注"[待确认]" |
| 会议后3天才发纪要 | 黄金时间是会议结束后2小时内 |

## 第2课：AI邮件写作 —— 每封邮件都像"老员工"

**时长**: 约60分钟 | **类型**: 实操为主

### 核心概念

职场邮件是职场第一印象。一封好的邮件三要素：

1. **清晰的目的**：收件人一眼就知道你要TA做什么
2. **恰当的语气**：对上级/平级/下级/客户的语气完全不同
3. **简洁的结构**：能100字说清楚不用200字

### AI邮件写作模板库

#### 模板1：向上汇报邮件

```text
你是一位资深职场沟通专家。请帮我写一封向上级汇报工作的邮件。

背景：
- 汇报事项：[如：本周项目进展汇报]
- 核心数据：[如：完成率达85%，比预期提前2天]
- 遇到的问题：[如：设计稿延迟交付，影响了前端排期]
- 需要的支持：[如：需要您协调设计团队本周五前完成最终稿]

邮件要求：
1. 标题：清晰概括邮件主题，20字以内
2. 开头：一句话说清写这封邮件的目的
3. 正文：先报喜（进展）再报忧（问题），每个问题后面跟一个"我的建议方案"
4. 结尾：用"需要您确认/支持的是……"明确收件人需要做什么
5. 语气：专业、自信、不卑不亢
6. 字数：控制在300字以内
```

#### 模板2：跨部门沟通邮件

```text
你是一位跨部门协作专家。请帮我写一封跨部门沟通邮件。

背景：
- 对方部门：[如：设计部]
- 沟通事项：[如：需要设计部配合在下周三前完成3张活动海报]
- 背景信息：[如：我们在做618大促活动，运营方案已经确认，现在就差设计素材]
- 可能遇到的阻力：[如：我知道设计部最近排期很满]

邮件要求：
1. 标题：用"协作"而非"请求"的语气
2. 开头：先感谢对方部门之前的工作/理解对方的忙碌
3. 正文：明确说清需要什么、为什么需要、截止日期是什么
4. 提供"降低对方负担"的方案（如：我们已准备好文案和参考图，设计只需排版）
5. 语气：合作型，"我们一起来做这件事"而非"你必须给我做"
6. 字数：控制在250字以内
```

#### 模板3：客户沟通邮件

```text
你是一位客户管理专家。请帮我写一封给客户的邮件。

场景：[选择一种]
- 场景A：项目节点汇报（向客户汇报项目进展）
- 场景B：问题沟通（项目出现延期/问题，需要和客户沟通）
- 场景C：方案确认（请客户确认方案并推进下一步）

具体背景：[填写你的具体情况]

邮件要求：
1. 体现专业度和对客户的重视
2. 如果是"问题沟通"，不要回避问题，但要同时给出解决方案
3. 用"我们"而非"我"——体现团队感
4. 结尾明确下一步行动和责任人
5. 字数：200-350字
```

#### 模板4：英文商务邮件

```text
You are a business English expert. Please help me write a professional email in English.

Context:
- Recipient: [e.g., a potential client / a senior executive / a cross-functional colleague]
- Purpose: [e.g., follow up on a meeting / introduce our product / request a meeting]
- Key points to include: [list 2-3 key points]
- Tone: [formal / semi-formal / friendly professional]

Requirements:
1. Subject line: clear and actionable
2. Opening: polite and contextual
3. Body: concise, 3-4 paragraphs max
4. Closing: clear call to action
5. Word count: 150-250 words
6. Also give me a Chinese translation for reference
```

## 第3课：AI数据分析 —— 不写代码也能做分析

**时长**: 约90分钟 | **类型**: 实操为主

### 动手实操：从数据到洞察的全流程

#### Step 1：数据理解与清洗（15分钟）

找一份你真实工作中可能遇到的数据（如销售数据、项目数据、用户数据），上传给AI：

```text
我上传了一份数据文件。请先帮我做一个全面的数据体检：

1. **数据概览**：
   - 共有多少行、多少列？
   - 每列的字段名、数据类型、含义猜测
   - 数据的时间范围（如果有日期列）

2. **数据质量**：
   - 哪些列存在缺失值？缺失比例是多少？
   - 哪些列存在明显的异常值或错误值？
   - 有没有重复的行？

3. **清洗建议**：
   - 哪些列需要清洗？怎么清洗？
   - 如果你能直接帮我清洗，请告诉我清洗后的变化

4. **关键字段的统计描述**：
   - 数值列：均值、中位数、最大值、最小值、标准差
   - 分类列：各类别的数量和占比
```

#### Step 2：探索性分析（25分钟）

```text
基于以上数据，请帮我做探索性分析：

1. **趋势分析**：如果有时间维度，随时间变化的趋势是什么？有没有明显的周期性或季节性？
2. **相关分析**：哪些变量之间可能存在关联？用通俗语言解释这些关联意味着什么
3. **分组对比**：按[关键分类维度，如：部门/地区/产品类型]分组，各组之间的差异是什么？
4. **异常发现**：数据中TOP 5的异常值是什么？可能的原因是什么？
5. **漏斗分析**（如果适用）：从开始到结束的转化率是多少？哪个环节流失最严重？

请用"非技术语言"解释每个发现，就像在给不懂数据的老板做汇报。
```

#### Step 3：AI生成可视化图表（20分钟）

```text
基于以上分析，请帮我设计图表方案：

1. 推荐最适合展示以上发现的5种图表类型
2. 对每个图表：
   - 说明它展示什么数据
   - 说明它想传达什么信息
   - 给出发到WPS Excel/腾讯文档中制作的具体步骤（点哪个按钮、选什么选项）
3. 如果可能，请生成Python代码，我可以复制到Google Colab中运行直接出图

图表类型建议：
- 趋势 → 折线图
- 对比 → 柱状图/条形图
- 占比 → 饼图/环形图/堆叠柱状图
- 分布 → 直方图/箱线图
- 关系 → 散点图/气泡图
- 构成 → 瀑布图/桑基图
```

#### Step 4：生成数据分析报告（20分钟）

```text
请基于以上所有分析，帮我生成一份完整的"数据分析报告"：

报告结构：
1. **报告摘要**（150字以内）：用3句话概括核心发现
2. **核心指标概览**：用表格展示关键指标
3. **五大发现**（每个发现含：标题 → 数据支撑 → 业务解读 → 行动建议）
4. **风险预警**：数据中暴露的2-3个风险点
5. **下一步建议**：基于数据，建议接下来做什么

格式要求：
- 适合直接粘贴到PPT或文档中
- 每个发现配一句"如果只能说一句话"的总结
- 所有数据保留一位小数
```

### 避坑指南

| ❌ 常见错误 | ✅ 正确做法 |
|-----------|-----------|
| 上传敏感数据（含客户信息） | 上传前做脱敏处理：真实姓名→"客户A"，手机号→"138****" |
| 让AI分析没有业务上下文的数据 | 提前告诉AI这些数据是什么业务场景下的 |
| 只看AI的分析，不看原始数据 | AI可能误读数据，关键发现要回到原始数据验证 |
| 图表越多越好 | 一个图表只讲一个故事，5张好图 > 15张凑数图 |
| 分析完了不行动 | 数据分析的终点不是"报告"，而是"决策" |

## 第4课：AI PPT制作 —— 10分钟从零到一份专业演示

**时长**: 约90分钟 | **类型**: 实操为主

### 核心概念

做PPT的传统痛点：70%的时间花在排版和找图上，只有30%在思考内容。AI PPT工具把这个比例倒过来——你花80%的时间打磨内容逻辑，AI花20%的时间执行排版。

### 避坑指南

## 第6课：AI文档写作 —— 周报、公文、需求文档样样精通

**时长**: 约90分钟 | **类型**: 实操为主

### 核心概念

职场写作的秘诀不是"文笔好"，而是**"结构清晰 + 信息密度高 + 读者友好"**。AI帮你解决"写不出来"的焦虑，你负责提供"写什么"的判断。

### 动手实操：AI职场写作全场景

#### 场景1：周报/双周报（15分钟）

```text
你是一位职场写作专家。请帮我写一份本周的工作周报：

基本信息：
- 姓名/岗位：[填写]
- 汇报对象：[填写]
- 本周日期：[填写]

本周工作内容（列出3-5条，每条一句话描述）：
1. [填写]
2. [填写]
3. [填写]

下周计划（列出3-5条）：
1. [填写]
2. [填写]

遇到的问题/需要的支持：[填写]

周报格式要求：
1. 用"本周进展 + 数据亮点 + 问题与思考 + 下周计划"四段式结构
2. 每条进展用动词开头，强调结果而非过程
3. 数据亮点用"数字+变化"的格式（如"用户增长12%，环比提升3个百分点"）
4. 问题部分先写"我的解决方案"，再写"需要你支持什么"
5. 总字数控制在300字以内
6. 语气：简洁、干货、不废话
```

#### 场景2：工作总结/述职报告（20分钟）

```text
你是一位职场写作教练。请帮我写一份工作总结：

- 工作周期：[如：2026年Q1 / 入职3个月 / 实习期]
- 岗位：[填写]
- 核心工作内容：[填写3-5条]
- 最骄傲的成果：[填写1-2个，尽量有数据]
- 不足与改进：[填写1-2条]
- 下阶段目标：[填写]

报告结构：
1. **工作概述**（100字）：一句话总结这个周期做了什么
2. **核心成果**（分3-4个小点，每点含：做了什么→结果是什么→数据支撑）
3. **亮点项目**（选1个最成功的项目展开：背景→我的角色→关键动作→成果）
4. **不足与反思**（坦诚但不过度否定自己，每个不足后面跟"改进措施"）
5. **下阶段计划**（目标+关键行动+预期成果）
6. **需要的支持**（具体、合理）

要求：
- 成果要有数据，没有精确数据用合理估计（标注"约"）
- 不写"我学到了很多"这种空话，写"我掌握了XX技能，在XX场景中应用了"
- 字数：800-1200字
```

8. **风险与假设**：我们需要验证的假设和主要风险

1. **需求概述**：用户故事 + 验收标准

## FAQ

**Q1：公司用的是钉钉/企业微信，不是飞书，怎么办？**

钉钉有"钉钉多维表"和"钉钉AI助理"，企业微信有"腾讯乐享AI"和"腾讯文档AI"。虽然工具不同，但工作流是一样的：录音→转写→AI提炼→纪要→待办。你可以用通义听悟（跨平台）作为折中方案，它不依赖特定办公平台。

**Q2：AI写的会议纪要，老板会发现是AI写的吗？**

高质量AI纪要是很难被发现的——因为纪要本身就是"结构化记录"，不需要"个人风格"。关键是：1）你必须人工审校，修改AI可能搞错的术语、人名、数据；2）在待办事项和风险判断上融入你自己的判断（AI不知道你的团队政治生态）。如果你只是把AI输出原封不动地贴出来，那确实会露馅，因为AI经常把"听起来合理"但"实际上不对"的东西写进去。

**Q3：Gamma生成的PPT只能导出图片，排版不够灵活？**

Gamma的弱项确实是精确排版。最佳实践：**用Gamma做"灵感稿"和"视觉参考"**，用WPS AI PPT或传统PPT做"最终交付稿"。具体流程：Gamma快速生成 → 确定视觉风格和内容框架 → 导出到WPS/PPT → 精细化排版。这样你既享受了Gamma的"快"，又不损失传统PPT的"精准"。

**Q4：AI做数据分析，和用Excel自己分析，哪个更可靠？**

AI分析的优势是"快速发现模式"，劣势是"可能过度解读噪声"。最佳实践：让AI做"探索性分析"（提假设、找模式），你自己做"验证性分析"（回到Excel确认关键发现）。AI分析 + 人工验证 = 既快又准。另外，AI分析之前一定要告诉AI你的数据是什么业务场景下的，否则AI可能会给出"技术上正确但业务上荒谬"的结论。

**Q5：用AI写周报算不算"偷懒"？领导会怎么想？**

用AI写周报不是偷懒，是效率。关键是：周报的"内容"——你做了什么、取得了什么成果——必须是你自己真实做到的。AI只是帮你把"做了A、B、C"表达得更清晰、更有结构。领导不会在意你是不是用AI写了周报，领导在意的是周报有没有信息量。一份AI辅助但内容充实的周报，远好于一份手写但空话连篇的周报。
