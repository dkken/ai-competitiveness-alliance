## 第1课：作品集思维 —— 为什么作品集比简历更重要？

**时长**: 约 90 分钟 | **类型**: 认知 + 规划

### 核心概念

#### 2026年招聘市场的三个变化

根据猎聘《2025 AI人才趋势报告》和 FrogHire.ai《2026 校园招聘白皮书》：

1. **简历筛选已进入 AI 时代**：超过 67% 的 500 强企业使用 ATS（申请人追踪系统）初步筛选简历。一份标准简历从被看到到被淘汰的平均时间是 **6 秒**。但一个作品集链接，HR 平均会点击并停留 **2-3 分钟**。

2. **"做过什么"正在取代"学过什么"**：2026 年应届生招聘中，74% 的 HR 表示"项目经验"是筛选候选人的第一要素，远超"学校排名"（31%）和"GPA"（19%）。—— 数据来源：中国青年报社《2026 应届生就业力调查》

3. **AI 协作能力成为通用技能标签**：越来越多的 JD 中出现"熟练使用 AI 工具""有 AI 协作项目经验"等要求。作品集是你证明这些能力的最佳载体。

#### 作品集 > 简历的三要素

| 要素 | 含义 | 反面案例 | 正面案例 |
|------|------|---------|---------|
| **真实性（Authenticity）** | 项目是你亲手做的，过程可追溯 | 网上复制粘贴别人的项目，一问三不知 | 每个项目都有你的思考、决策、迭代过程 |
| **相关性（Relevance）** | 项目与你目标岗位有直接关联 | 申请市场营销岗，作品集全是写代码 | 申请市场营销岗，作品集有用户调研、竞品分析、营销方案 |
| **可展示性（Presentability）** | 项目能以直观方式呈现给非专业人士 | 只有代码仓库，没有说明文档和截图 | 有完整的项目介绍、过程截图、数据对比、个人反思 |

#### 面试官看作品集时，实际在找什么？

根据对 50+ 位 HR 和业务面试官的访谈（FrogHire.ai 2026 报告），他们看作品集时关注以下 5 个问题：

1. **这个人能不能独立完成工作？**（看项目完整度）
2. **这个人的思考深度如何？**（看项目反思和迭代过程）
3. **这个人的学习能力怎样？**（看技术栈/工具链的多样性）
4. **这个人有没有"产品思维"？**（看是否从用户/业务角度出发）
5. **这个人适合我们团队吗？**（看沟通风格、审美偏好、价值观）

#### Vibe Coding 核心思维

Vibe Coding（氛围编程）是 2025 年由 Andrej Karpathy 提出的概念，2026 年已成为主流开发范式。核心原则：

1. **你描述需求，AI 生成代码** —— 你不需要写一行代码
2. **迭代式改进** —— 看到效果后，用自然语言告诉 AI 要改什么
3. **拥抱错误** —— AI 生成的代码可能有 bug，把错误信息复制给 AI，它通常会自己修复
4. **你负责决策** —— 技术选型、设计方向、产品逻辑由你决定，AI 负责执行

### 动手实操：用 Bolt.new 搭建个人网站

以下是用 **Bolt.new**（2026 年最推荐的零代码建站工具）搭建个人网站的全流程。

#### 第一步：注册与准备（10 分钟）

1. 打开 [bolt.new](https://bolt.new)
2. 使用 GitHub 账号登录（推荐）或邮箱注册
3. 进入主界面，你会看到一个类似 ChatGPT 的对话框

#### 第二步：生成网站初版（15 分钟）

在 Bolt.new 的对话框中输入以下 Prompt（根据你的实际情况修改）：

```text
请帮我创建一个个人作品集网站，要求如下：

**基本信息**：
- 姓名：[你的名字]
- 专业：[你的专业]
- 求职方向：[你的目标岗位]
- 一句话介绍：[用一句话概括你的核心优势]

**网站结构**（单页滚动式）：
1. 顶部导航栏：Logo（名字缩写）+ 导航链接（首页、项目、关于、联系方式）
2. Hero 区域：大标题 + 一句话介绍 + 两个按钮（"查看作品"和"下载简历"）
3. 关于我：一段 2-3 句话的自我介绍 + 3-5 个技能标签
4. 项目展示：3-5 个项目卡片，每个包含：项目截图、项目名称、简短描述、技术标签、查看详情按钮
5. 联系方式：邮箱、GitHub 链接、LinkedIn（如有）
6. 页脚：版权信息

**设计要求**：
- 现代简约风格，参考 Linear / Notion 的设计语言
- 配色方案：主色调为深蓝/紫色系（#6366f1 或类似），干净的白底
- 字体：使用系统默认字体（苹果用 SF Pro，Windows 用 Segoe UI）
- 响应式：适配手机和平板
- 微交互：按钮 hover 效果、卡片 hover 上浮、滚动平滑

**技术栈**：React + Tailwind CSS
```

#### 第三步：迭代优化（30 分钟）

Bolt.new 生成初版网站后，你会看到实时预览。接下来用自然语言逐步优化：

**迭代 1：调整 Hero 区域**
```text
Hero 区域太高了，缩短到屏幕高度的 70%。背景加一个微妙的渐变，从深蓝到紫色。大标题字体再大一些，加粗。
```

**迭代 2：优化项目卡片**
```text
项目卡片改成 3 列网格布局。每个卡片加一个 placeholder 图片（可以用 emoji 或渐变色块代替）。鼠标悬停时卡片上浮 8px 并加阴影。
```

**迭代 3：添加技能标签**
```text
在"关于我"区域的技能标签中，给每个标签加一个小图标（emoji 即可）。标签颜色改为浅紫背景 + 深紫文字。
```

**迭代 4：添加导航滚动**
```text
顶部导航栏加一个"回到顶部"按钮。点击导航链接时平滑滚动到对应区域。当页面滚动超过 100px 时，导航栏背景变为半透明白色毛玻璃效果。
```

**迭代 5：添加暗色模式**
```text
添加一个暗色模式切换按钮，放在导航栏右侧。使用 Tailwind 的 dark mode 类。默认跟随系统主题。
```

::: tip 迭代技巧
- 每次只提一个改动，不要一次说太多
- 如果 AI 理解错了，把错误描述清楚，附上当前效果截图（如果可能）
- 如果 AI 改坏了，说"请撤销刚才的改动，恢复到上一版"
- 多使用"参考 XXX 网站"来传达设计意图，AI 能理解这些参考
:::

#### 第四步：填入真实内容（25 分钟）

网站框架搭建好后，开始替换占位内容为你自己的真实内容：

```text
请帮我修改 Hero 区域的文字：
- 大标题改为："用AI重新定义[你的专业]"
- 一句话介绍改为："[你的专业]学生 | AI工具深度使用者 | 已完成[X]个AI协作项目"
- "查看作品"按钮链接到项目展示区域
```

## 第3课：作品集内容创作

**时长**: 约 90 分钟 | **类型**: 实操 + 内容创作

### 核心概念

#### 项目展示标准结构

每个项目都应该遵循"背景 → 目标 → 方法 → 成果 → 反思"的五段式结构：

```
┌────────────────────────────────────────────────────────────┐
│                    项目展示标准结构                          │
├──────────┬─────────────────────────────────────────────────┤
│  背景    │ 为什么做这个项目？遇到了什么问题？有什么机遇？       │
│  Context │ 1-2 句话交代项目缘起                              │
├──────────┼─────────────────────────────────────────────────┤
│  目标    │ 你想达成什么？具体的、可衡量的目标                  │
│  Goal    │ 例如："提升用户活跃度 20%"而非"让产品更好"         │
├──────────┼─────────────────────────────────────────────────┤
│  方法    │ 你做了什么？用了什么工具？有什么关键决策？           │
│  Method  │ 这是展示你思考过程的核心部分                        │
├──────────┼─────────────────────────────────────────────────┤
│  成果    │ 达成了什么效果？有数据吗？有反馈吗？                 │
│  Result  │ 尽量用数据说话，没有数据就用对比或用例              │
├──────────┼─────────────────────────────────────────────────┤
│  反思    │ 如果重来一次会怎么做？学到了什么？                   │
│  Reflect │ 这是展示你学习能力和成长心态的关键                  │
└──────────┴─────────────────────────────────────────────────┘
```

#### 为什么"反思"是作品集最重要的部分？

面试官看作品集时，最关注的是你的**思考过程**，而不是最终结果。原因很简单：

- 最终结果可能受限于资源、时间、外部因素，不全是你的能力体现
- 但你的**反思能力**——认识到哪里做得不够好、为什么这样选择、学到了什么——直接反映了你的成长潜力

**一个反面例子**：
> "我完成了这个项目，效果很好，用户反馈积极。"

**一个正面例子**：
> "项目完成后，我意识到最初的设计假设（用户更喜欢 A 方案）是错误的。通过 A/B 测试数据，我发现 B 方案的用户留存率高出 35%。如果重来一次，我会在项目初期就进行用户调研，而不是凭直觉做决策。这个经验让我在后续项目中始终把'数据验证'放在'主观判断'之前。"

### 动手实操：AI 优化作品集文案

#### 第一步：撰写项目文案初稿（20 分钟）

选择一个你已完成的项目，按照五段式结构写出初稿。不需要写得很好，把想法写出来即可。

#### 第二步：用 AI 优化文案（20 分钟）

将你的初稿粘贴到 AI 中，使用以下 Prompt：

```text
你是一位资深文案策划，专门帮助求职者优化作品集文案。请帮我优化以下项目描述。

项目背景：[你的背景描述]
项目目标：[你的目标描述]
方法过程：[你的方法描述]
项目成果：[你的成果描述]
个人反思：[你的反思描述]

优化要求：
1. 保持原来的信息不变，但让表达更专业、更有冲击力
2. 使用 STAR 法则（Situation-Task-Action-Result）增强逻辑性
3. 突出"我做了什么"而非"团队做了什么"
4. 加入具体的数据和细节（如果有）
5. 每个部分控制在 3-5 句话，避免冗长
6. 语气自信但不夸张，适合应届生身份

请给出优化后的完整文案，并在每个部分标注优化要点。
```

#### 第三步：数据可视化（20 分钟）

如果有数据，用 AI 帮助你生成图表：

```text
我有以下项目数据，请帮我设计数据可视化方案：

[粘贴你的数据]

请提供：
1. 最适合展示这些数据的图表类型（柱状图/折线图/饼图/雷达图等）
2. 用 Python（matplotlib/seaborn）或 HTML/CSS 生成可视化图表
3. 图表配色建议（与我的网站配色协调）
4. 每个图表的一句话解读

如果数据量不大，请直接生成 HTML/CSS 的可视化代码，我可以嵌入到作品集网站中。
```

#### 第四步：生成项目缩略图（20 分钟）

每个项目卡片需要一张吸引人的缩略图。如果你没有设计能力，可以用 AI 生成：

```text
请帮我生成一张项目缩略图的设计方案。

项目名称：[项目名称]
项目类型：[如：数据分析报告 / 竞品分析 / 用户调研]
核心关键词：[3-5个关键词]
配色方案：与我的网站配色协调（主色 #6366f1，背景白/浅灰）

请提供：
1. 用 HTML/CSS 生成一个简洁的缩略图（包含项目标题、类型标签、关键词）
2. 或者用 SVG 生成一个抽象的几何图形作为背景
3. 尺寸：1200×630px（适合社交媒体分享）
```

### 作品集内容质量检查清单

在发布前，用以下清单逐项检查每个项目：

| 检查项 | 标准 | ✓ |
|--------|------|---|
| 背景清晰 | 读者能理解为什么做这个项目 | |
| 目标具体 | 不是空泛的"提升能力"，而是可衡量的具体目标 | |
| 方法详实 | 展示了具体工具、步骤、决策过程 | |
| 成果有力 | 有数据或具体反馈，不是"感觉不错" | |
| 反思真诚 | 指出了不足和改进方向，展示了成长 | |
| AI 角色透明 | 清楚标注了 AI 在项目中扮演的角色 | |
| 无错别字 | 通读一遍，没有低级错误 | |
| 手机可读 | 在手机上打开网站，内容是否正常显示 | |

## 第4课：专业方向作品集选题指南

**时长**: 约 90 分钟 | **类型**: 规划 + 实操

### 核心概念

#### AI 辅助项目全流程

无论你是什么专业，一个完整的 AI 协作项目通常遵循以下流程：

```
选题 → 调研 → 执行 → 展示
  │       │       │       │
  │  AI   │  AI   │  AI   │  AI
  │ 帮分析 │ 帮收集 │ 帮执行 │ 帮优化
  │ 可行性 │ 和整理 │ 和迭代 │ 和呈现
  │       │       │       │
  └───────┴───────┴───────┘
```

**关键是**：在每个阶段都记录你的思考过程，这是作品集最有价值的部分。

#### 如何在项目中展示"思考"而非"做法"

| 阶段 | 做法展示（平庸） | 思考展示（优秀） |
|------|---------------|---------------|
| 选题 | "我选择了分析新能源汽车行业" | "对比了 3 个备选方向后，我选择新能源汽车行业，因为：① 数据可获取性高；② 与我的目标岗位（行业研究）强相关；③ 2026 年固态电池技术突破带来新的分析视角" |
| 调研 | "我在网上搜索了相关资料" | "我用了 3 种方式收集数据：① 上市公司财报（一手数据）；② AI 辅助分析行业报告（二手数据）；③ 用户评论情感分析（验证数据）。3 种数据交叉验证，确保结论可靠" |
| 执行 | "我用 AI 帮我写了分析报告" | "我先让 AI 列出了分析框架（PEST + 波特五力），然后补充了我自己的判断。在 AI 生成的分析中，我发现 3 处数据错误，手动修正后重新验证。整个过程中 AI 承担了约 60% 的工作量，但我负责了所有关键判断" |
| 展示 | "我把报告放在了网站上" | "报告完成后，我意识到纯文字的可读性差。于是我用 AI 生成了 5 张数据可视化图表，并写了一个 300 字的核心发现摘要放在开头。一份 5000 字的报告，现在 30 秒就能抓住关键信息" |

## FAQ

**Q1: 我没有可以展示的项目怎么办？**

A: 这正是本模块要解决的问题。第4课提供了 8 个专业方向、32 个具体选题，选择 2-3 个在课后完成。每个项目平均耗时 6-10 小时，一周内可以完成。关键是：不要追求完美，先完成再迭代。

**Q2: 我不会写代码，真的能搭建网站吗？**

A: 完全可以。Bolt.new 和 Lovable 这两个工具在 2026 年已经非常成熟，你只需要用自然语言描述你想要的效果，AI 会自动生成完整的网站代码。本模块第2课提供了完整的 Prompt 模板和迭代流程，跟着做就行。

**Q3: 我的项目看起来不够"专业"怎么办？**

A: 应届生的作品集不需要和行业专家比。面试官看的是：① 你有没有动手能力；② 你的思考过程是否清晰；③ 你的学习态度和成长潜力。一个"不完美但有反思"的项目，远胜于一个"完美但看不出你的贡献"的项目。

**Q4: 作品集应该包含多少个项目？**

A: 3-5 个精品项目比 10 个草草了事的项目好得多。建议：2 个深度项目（展示专业深度）+ 2 个广度项目（展示工具应用能力）+ 1 个个人特色项目（展示你的独特兴趣）。

**Q5: 我的 AI 使用比例很高，会不会被认为"不是自己做的"？**

A: 2026 年的招聘市场已经接受了"AI 协作"这一事实。关键是：① 清楚标注 AI 在项目中的角色；② 展示你独立做出的关键决策；③ 反思 AI 的局限性和你如何克服。把"AI 使用比例高"变成"AI 协作能力强"的证明，而不是"能力不足"的证据。

**Q6: 建站工具选哪个？Bolt.new vs Lovable vs Cursor？**

A: 零基础 → Bolt.new（最易上手，一键部署）。追求设计质量 → Lovable（UI 最精美）。想学代码 → Cursor（最灵活，但需要学习成本）。建议先用 Bolt.new 快速搭建，后续如果想深入定制，再迁移到 Cursor。
